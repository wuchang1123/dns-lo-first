# Local First - 本地优先 DNS 服务器

基于 Go 开发的高性能 DNS 服务器，支持智能分流、病毒检测和定时更新。

- IPv4 Only，AAAA 查询转发，本服务不处理
- 终端里按 Ctrl+C 退出，停止 DNS 服务器

## 上游服务器

- 划分为本地和海外两大类
    - 各类按职能分流规则，需要查询的时候先随机取一个 IP 查询，失败时并发尝试剩下同类所有上游，这些上游再都失败的话才视为失败
- 支持IP
- 支持DoH
- 实现上游服务器打分制
    - 最近响应延迟低的分值高
    - 按分值排序使用上游服务器
    - 分数相等的上游服务器随机提取
- 上游服务器超时处理
    - 冻结1分钟
    - 同类型全部冻结时先解冻最早冻结的上游

## 缓存

- 基础响应缓存：
    - 负责主要的 DNS 缓存
    - 采取乐观缓存，过期时先使用缓存返回响应（ttl=2，后台随时间减少，0后再有请求重置2起计循环），后台刷新缓存
    - 缓存 key 按 qname + qtype + qclass
    - 每天定时清除早于 48 小时前创建的条目
    - 需要写上对应响应的上游服务器
- 校对结论缓存：
    - 保存域名的本地和海外响应信息
    - 保存对应上游服务器信息
- 基础缓存+校对缓存：同时内存和文件存储，内存加速，文件持久化
    - 内存快速响应，文件人可读
- 基础缓存和校对缓存各个一套，相互不干扰

## 文件

- 配置文件 @config.yaml
- GFW域名 @data/common_blocked_domains.txt，有注释、空行
- ASN信息 @data/domain_asn.yaml 域名/组织/IP段映射信息

## 智能分流

- 默认所有域名使用本地和海外 DNS 上游服务器，并发查询
- 配置文件 local_domains 定义的域名只使用本地 DNS 上游服务器
- 配置文件 upstream.local_only 定义的域名只使用本地 DNS 服务器
- 配置文件 upstream.overseas_only 定义的域名只使用海外 DNS 上游服务器
- overseas_only > local_only > local_domains > 默认并发
    - apex + 子域
    - 支持 *.example.com 严格子域（RFC 4592）
- 所有地域名通过配置文件 local_domains.source_url 定时更新
    - 更新后可热加载

## 病毒识别系统

1.所有上游服务器的响应都进行简单的 ASN 病毒识别
    - 配置文件有相应开关
    - ASN信息 domain_asn.yaml 域名/组织/IP段映射信息
    - 默认响应的 IPs 都没病毒
    - domain_asn.yaml 中有定义的域名
        - 遍历响应 IPs，如果有 IP 在该域名对应组织的 IP 段内即视为命中
            - 遍历完成后如果至少有一个命中，就返回所有命中的 IPs 给客户端，后台继续运行后续逻辑，或后台继续等另一个上游如果有的话
        - 否则该上游响应视为可疑，不直接返回，继续等待其他上游或进入后续校验逻辑
    - domain_asn.yaml 中没有定义的域名视为可疑，继续后续逻辑
2.所有上游服务器的响应后，如果没有通过 ASN 验证，并且是重点嫌疑域名则启动 TSL 证书校验
    - NXDOMAIN、无 A 记录这类结果直接跳过 TLS校验
    - TLS 证书校验（用查询域名作为 SNI，连接响应 IP 的 443，校验证书链和 SAN，需要支持CNAME）
    - TLS 校验不能阻塞所有提前返回响应的情况
    - 证书校验多个 IP 中任一通过视为校验通过
    - 校验通过，立即返回本地响应结果，记录基础响应缓存
    - 打印日志，至少包含域名、上游，耗时的信息
3.需要使用本地和海外 DNS 上游服务器并发查询的域名
    - 没有校对缓存的时候
        - 无论哪类上游响应通过 ASN 识别，都立即返回（ttl=2，后台随时间减少，0后再有请求重置2起计循环），后台继续运行后续逻辑，或等另一个上游如果有的话
        - 无论哪类上游响应通过 TLS 校验，都立即返回，并且取消其它并发逻辑
        - 若有TLS 校验的话等 TLS 校验完成，没有就等待本地和海外 DNS 响应，最多等待6s（配置项server.concurrent_timeout）
            - 都有返回的话
                - 如果本地和海外结果一致，取海外响应记录基础响应缓存
                    - 结果一致条件
                        - 完全一样，或者有一个IP一样
                        - 有IP在相同的IP段内：先校验"/24"，失败再校验"/16"
                - 如果结果不一致
                    - 响应统一处理：ttl=2，后台随时间减少，0后再有请求重置2起计循环
            - 6s超时
                - 有海外响应结果，以该结果响应回客户端，并且记录基础响应缓存
                - 没有海外响应结果，以本地响应结果（若无则空）
                - 响应统一处理：ttl=2，后台随时间减少，0后再有请求重置2起计循环

## 日志系统

- 日志时间保留显示精确到ms
- 日志时间的年份只取最后两位

### 基础日志
    - 日志等级: debug, info, warn, error, fatal
    - 日志采用时区：Asia/Shanghai
    - 详细记录日志，有足够信息方便调试
    - 每天0点清除超过 48 小时的日志（启动时不要）

### 性能日志
    - 耗时统一ms单位，取小数点3位，四舍五入
    - 日志格式前面统一先显示时间、耗时
    - 耗时记录（独立文件记录 query.log）
        - 打印每个查询从进入到返回所需时间
            - 包含 qname/qtype/qclass、耗时、上游服务器、响应 IP 和 rcode
                - 耗时排在时间后面，域名前
                - 无错误不输出 rcode
                - 耗时格式
                    - 总耗时（本地上游服务器响应耗时若有 + 海外上游服务器响应耗时若有 + TLS 证书校验耗时）
            - 无键值格式输出
        - 打印每个上游查询从进入到返回所需时间
            - 最少包含上游服务器信息、开始&结束时间（HH:MM:SS.mmm）
        - 打印每个TSL校验从进入到返回所需时间
            - 最少包含域名、IP 信息、开始&结束时间（HH:MM:SS.mmm）
    - 每天0点清除超过 48 小时的日志（启动时不要）

## Bootstrap DNS

- 配置文件 bootstrap_dns，下载配置解析域名时使用的递归 DNS（Do53，host:port）
    - 本服务刚启动时系统 resolv 可能不可用（如指向 [::1]:53），此处显式指定可避免依赖本机解析
    - 留空则使用配置中的上游 DNS 服务器；若仍为空则内置 223.5.5.5 / 8.8.8.8 / 1.1.1.1。
